Detailed Description

In Domain Driven Design, an Entity is An object that is not defined by its attributes, but rather by a thread of continuity and its identity. So for example, a user is an entity because you can change the user’s email address without changing the identity of the user. This is because the user has an id so even when you change the user’s attributes, it’s still the same user. So an Entity is basically just a unique object within the system that has identity. You can’t have two users with the same id and so users are Entities. An entity can be a user, an order, a product, a blog post, really anything with an identity.
